+ Was ist die Startrampe?

Die Startrampe ist ein interaktives Werkzeug, das dich Schritt für Schritt vom leeren Projektordner bis zur fertigen Live-Website führt. Du gibst dein Projekt ein — die Startrampe generiert dir eine personalisierte Anleitung mit allen Befehlen, die du nur noch kopieren und ausführen musst.

Keine Vorkenntnisse im Deployment nötig. Kein Backend, keine Datenbank, kein Abo. Alles läuft direkt in deinem Browser.

🎯

Personalisiert

Jeder Befehl enthält deinen Projektnamen, GitHub-User und deine Subdomain.

📋

Copy und Paste

Alle Terminal-Befehle haben einen Kopier-Button — einfach einfügen und ausführen.

🔒

Privat

Nichts verlässt deinen Browser. Kein Account, kein Tracking, keine Daten auf Servern.

⚡

In 15 Minuten live

Vom leeren Ordner bis zur fertigen Subdomain — in einer Kaffeepause erledigt.

+ Was brauchst du?

Bevor du die Startrampe nutzt, richte diese drei kostenlosen Konten ein. Falls du schon welche hast, überspringe die jeweiligen Punkte.

  • 1 GitHub-Konto — Hier wird dein Code gespeichert. GitHub ist kostenlos und braucht nur eine E-Mail-Adresse. Konto erstellen →
  • 2 Vercel-Konto — Vercel stellt deine Website ins Internet. Am einfachsten: Mit deinem GitHub-Konto anmelden. Konto erstellen →
  • 3 Cloudflare-Konto — Cloudflare verwaltet deine Domain und leitet Besucher zu deiner Website. Ebenfalls kostenlos. Konto erstellen →
  • 4 Git installiert — Ein Versionskontrollsystem, das auf den meisten Macs schon vorinstalliert ist. Teste mit git --version im Terminal. Herunterladen →
  • 5 Eine eigene Domain — Deine Subdomain wird unter deiner Domain eingerichtet (z. B. mein-projekt.deine-domain.de).
+

Alle Dienste sind in der kostenlosen Stufe völlig ausreichend. Du brauchst nirgends ein Bezahl-Abo.

Stelle sicher, dass du Konten bei GitHub, Vercel und Cloudflare hast, Git installiert ist und du eine eigene Domain besitzt, die in Cloudflare verwaltet wird.

+ So funktioniert es

Die Startrampe führt dich durch 6 Schritte. Du füllst ein kurzes Formular aus — den Rest erledigt die Startrampe für dich.

1

Eingaben

Beschreib dein Projekt, gib deinen GitHub-Benutzernamen ein und wähle dein Betriebssystem und deine Entwicklungsumgebung. Aus deiner Beschreibung werden Projektname und Subdomain automatisch abgeleitet.

2

Projekt lokal anlegen

Du bekommst die passenden Befehle für dein System — Ordner erstellen, Editor öffnen, erste index.html anlegen. Alles zum Kopieren bereit.

3

Git und GitHub

Git wird initialisiert, dein Code committet und ein neues GitHub-Repository angelegt. Die Push-Befehle enthalten bereits deinen Benutzernamen und Repo-Namen.

4

Vercel-Deployment

Du importierst dein GitHub-Repo in Vercel. Ein Klick auf „Deploy" — und deine Seite ist unter einer Vercel-Preview-URL erreichbar.

5

Custom Domain einrichten

Du fügst deine Wunsch-Subdomain in Vercel hinzu und setzt einen CNAME-Eintrag in Cloudflare. Die Startrampe zeigt dir genau, welche Werte wohin gehören.

6

Live!

SSL-Zertifikat wird automatisch erstellt. Nach wenigen Minuten ist dein Projekt unter deiner Subdomain erreichbar. Jeder zukünftige git push aktualisiert die Seite automatisch.

+ Was du eingeben musst

Die Startrampe braucht nur vier Angaben von dir. Alles andere wird automatisch abgeleitet.

  • ✓ Projektbeschreibung — Ein bis zwei Sätze, z. B. „Ein Kalorien-Rechner für meine Ernährungsberatung". Daraus leitet die Startrampe den Projektnamen ab.
  • ✓ GitHub-Benutzername — Dein Login-Name auf GitHub, z. B. maxmustermann.
  • ✓ Betriebssystem — macOS, Windows oder Linux. Bestimmt die Terminal-Befehle.
  • ✓ Entwicklungsumgebung — Claude Code, Cursor, VS Code oder Terminal. Bestimmt die Arbeitsschritte.
+

Projektname, Subdomain und Repo-Name werden automatisch aus deiner Beschreibung abgeleitet. Du kannst sie jederzeit manuell anpassen, bevor du die Anleitung generierst.
